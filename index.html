What we're building

Designed for clarity, built for speed.

Every layer of Noryx OS is purpose-built — from the kernel to the compositor to the AI pipeline.

01

Calm Glass

A compositing layer that reduces visual noise. Adaptive contrast, muted chrome, and a default dark system theme that feels effortless.

02

Snappy

Cold boot <10 seconds. App launch under 400ms. Zero background bloat. Noryx runs lean because the base image is read-only and minimal.

03

Immutable

OSTree-based root filesystem. Updates are atomic — the whole image swaps in one operation. Rollback to any previous state instantly.

04

Noryx.AI

A system-native AI assistant. Local 7B LLM with GPU acceleration (CUDA, ROCm, Vulkan). Context-aware. No data leaves your machine by default.

05

Compatibility

Native Linux & Flatpak apps first. Android via Waydroid. Windows via Wine/Proton. Cloud fallbacks last. Most mainstream apps work on day one.

06

NoryxKit SDK

Build apps that talk to the AI layer, respond to system context, and ship as Flatpaks. Open SDK, permissive license, full documentation.

Vision

"The best OS is the one you never think about — it just amplifies what you're already trying to do."

Most operating systems are infrastructure you fight. Updates interrupt. Telemetry leaks. AI is bolted onto interfaces designed in 2001. Noryx starts from a different premise: the OS should disappear and let intelligence surface naturally.

We're not building another Linux distro with a theme pack. We're building a new system layer — a compositor, an AI runtime, and a design language that treats calm, speed, and privacy as non-negotiable defaults.

Zero default telemetry. Your data stays on your machine. Period.

About

Why we started.

"Every OS I used felt like it was built for someone else's workflow. I wanted one that felt like it was built for mine — calm, fast, and smart enough to stay out of the way."

NORYX started as a frustration. Modern operating systems are bloated, noisy, and architecturally stuck in the past. AI was being bolted onto 25-year-old interfaces instead of being woven into the system itself.

We're a small team of systems engineers, AI researchers, and designers. We believe the next operating system won't come from patching the old ones — it has to be built from scratch with intelligence, privacy, and calm as first principles.

Noryx is that project. An AI-native OS — not a distro, not a theme, not a wrapper. A new foundation for personal computing.

FAQ

Common questions.

Everything you need to know about Noryx OS before the developer preview.

Year 1 targets x86-64 laptops and desktops with good Linux driver coverage — Framework Laptop 13/16, ThinkPad T/X series, Dell XPS 13/15, and similar. ARM support (Qualcomm/MediaTek reference boards) comes in Year 2.

Yes. Noryx follows a compatibility priority: Native Linux/Flatpak apps first (VS Code, Chrome, Discord, Spotify, Steam), then Android apps via Waydroid, then Windows apps via Wine/Proton, then cloud fallbacks. Most mainstream apps work on day one.

On-device by default. Noryx.AI runs a quantized local LLM (7B parameters) with full GPU acceleration via CUDA, ROCm, and Vulkan. No data leaves your machine unless you explicitly opt into a cloud connector. Zero default telemetry.

The Developer Preview (Month 9, Q4 2026) will go to a closed cohort of early sign-ups first. Join the waitlist to be in the first wave. A public open beta follows in Month 10–12.

Core OS components — the launcher, Copilot CLI, and design language — are open-sourced under MPL 2.0. Proprietary UX layers, model weights, and first-party app assets are retained in-house. The SDK is open with a permissive license.

Noryx uses an OSTree-based immutable base image. Updates are atomic — the entire OS image is swapped in one operation. If anything goes wrong, you roll back to the previous image instantly. Updates stage in the background and apply on reboot.
