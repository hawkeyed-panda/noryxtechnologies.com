What we're building

Five pillars. One coherent system.

Every element of Noryx OS is designed around one promise: everything you do, faster and calmer.

01

Calm Glass design language

Soft depth, subtle motion, 60–120Hz friendly. Large targets, edge gestures, and a command palette that replaces menu hunting forever.

02

Snappy to everything

Under 200ms to anything visible. Under 1s cold-start on SSD. Enforced by CI performance budgets — regressions block release.

03

Immutable and atomic

OSTree-based immutable base, atomic updates, and instant rollback. Btrfs snapshots, SELinux enforcing, sandboxed Flatpak portals.

04

Noryx.AI — private, on-device

Local LLM runner with GPU paths (CUDA, ROCm, Vulkan). Hold-to-talk from anywhere. All data stays on your machine by default.

05

Runs everything, day one

Linux Flatpaks, Android via Waydroid, Windows via Wine/Proton, and cloud fallbacks. Native → Android → Wine → VM, always in that order.

The vision

The first calm, AI-native desktop that runs your apps — and gets out of your way.

Today's operating systems treat AI as an add-on — a chatbot layered over decades-old interface patterns. Noryx rethinks the whole stack. Intelligence is woven into files, notifications, settings, search, and every app from day one.

Most users have accepted that computers demand adaptation. They hunt through menus, manage fragmented workflows, and patch together apps that don't talk to each other. Noryx inverts this contract — the system adapts to you, surfacing what you need when you need it.

We're starting with x86 laptops and desktops — the hardware where driver coverage is deepest. ARM comes in Year 2. A broader Noryx ecosystem — Noryx Phone, Slab, and PC — follows.

The origin

Why Noryx exists.

"Most modern operating systems still follow interaction models designed decades ago. Even as AI became incredibly capable, it mostly existed as an add-on — sitting on top of old assumptions rather than being woven into the core."
Join the waitlist

Noryx OS is an AI-native operating system being built to reimagine the way we use our devices. The idea is to move beyond traditional UI-first operating systems and design a system where intent, context, and intelligence are foundational — not add-ons.

It started from a simple observation: windows, files, folders, and command interfaces have barely changed since the 1980s. AI has become extraordinarily capable, yet it mostly exists as an isolated application. We explored what happens when intelligence is embedded into the core interaction model itself — intent parsing, system-level orchestration, and adaptive UI layers built in from day one.

To prove the concept, a Flutter-based prototype shell was designed and built from scratch — a floating AI interaction layer that acts as the primary system interface, context-aware command execution across apps and files, a custom UI system with intent-driven controls, and early exploration of how a small language model could interpret system-level commands.

The process of designing and building that prototype — thinking deeply about OS architecture, interaction paradigms, and how AI could fundamentally reshape human-computer interaction — became the foundation for everything Noryx is building now.

FAQ

Common questions.

Everything you need to know about Noryx OS before the developer preview.

What hardware will Noryx OS run on?

Year 1 targets x86-64 laptops and desktops with good Linux driver coverage — Framework Laptop 13/16, ThinkPad T/X series, Dell XPS 13/15, and similar. ARM support (Qualcomm/MediaTek reference boards) comes in Year 2.

Can I run my existing apps on Noryx OS?

Yes. Noryx follows a compatibility priority: Native Linux/Flatpak apps first (VS Code, Chrome, Discord, Spotify, Steam), then Android apps via Waydroid, then Windows apps via Wine/Proton, then cloud fallbacks. Most mainstream apps work on day one.

Is the AI on-device or cloud-based?

On-device by default. Noryx.AI runs a quantized local LLM (7B parameters) with full GPU acceleration via CUDA, ROCm, and Vulkan. No data leaves your machine unless you explicitly opt into a cloud connector. Zero default telemetry.

When can I try it?

The Developer Preview (Month 9, Q4 2026) will go to a closed cohort of early sign-ups first. Join the waitlist to be in the first wave. A public open beta follows in Month 10–12.

Is Noryx OS open source?

Core OS components — the launcher, Copilot CLI, and design language — are open-sourced under MPL 2.0. Proprietary UX layers, model weights, and first-party app assets are retained in-house. The SDK is open with a permissive license.

How does Noryx handle updates and system stability?

Noryx uses an OSTree-based immutable base image. Updates are atomic — the entire OS image is swapped in one operation. If anything goes wrong, you roll back to the previous image instantly. Updates stage in the background and apply on reboot.
